## 修复statistics.ts相关API调用问题

### 问题分析
后端已实现所有statistics.ts中定义的API端点，但前端页面调用这些接口时无法获取数据。可能的原因包括：
1. 数据格式不匹配：前端期望的字段名与后端返回的不一致
2. API调用方式不正确
3. JWT验证问题
4. 前端组件未正确处理返回数据

### 修复方案

#### 1. 修复statistics.ts中的数据处理逻辑
- 确保前端能正确处理后端返回的数据格式
- 添加更详细的错误日志记录
- 统一处理响应数据的结构

#### 2. 检查并修复JWT验证机制
- 确保JWT token正确传递和验证
- 修复可能存在的token格式问题

#### 3. 优化API调用错误处理
- 添加更友好的错误提示
- 确保错误信息能正确传递给用户

#### 4. 修复前端组件中的数据使用
- 确保组件能正确调用API
- 修复数据绑定和渲染问题

### 具体修改步骤

1. **修改statistics.ts**：
   - 更新所有API函数，确保能正确处理后端返回的ResultMap格式
   - 添加更详细的错误日志
   - 统一数据字段映射

2. **检查http.ts**：
   - 确保响应拦截器能正确处理后端返回的ResultMap格式
   - 修复可能存在的响应处理逻辑问题

3. **测试API调用**：
   - 使用浏览器开发者工具检查API请求和响应
   - 验证token是否正确传递
   - 检查响应数据格式是否符合前端期望

4. **修复前端组件**：
   - 确保组件能正确调用API
   - 修复数据绑定和渲染问题

### 预期效果
- 前端页面能成功获取并显示统计数据
- API调用错误能得到友好提示
- JWT验证机制正常工作
- 数据格式统一，前端能正确处理

### 测试方法
1. 启动前后端服务
2. 登录系统，进入相关页面
3. 检查浏览器控制台是否有API调用错误
4. 验证统计数据是否正确显示
5. 测试不同角色（用户、医生）的统计数据显示

### 风险评估
- 低风险：所有修改都在现有代码基础上进行，不涉及核心架构变更
- 可回滚：修改的文件都可以通过版本控制系统回滚
- 影响范围：仅影响统计数据显示功能

### 依赖关系
- 后端服务正常运行
- 数据库中有测试数据
- JWT token生成和验证正常工作
