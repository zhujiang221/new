# 宠物医院管理系统 - Figma原型设计需求文档

## 系统概述
宠物医院管理系统，包含用户端和医生端，每个端都需要支持移动端和电脑端两种设备类型。

## 三、设计规范要求

### 布局要求
- **移动端**：单列布局，底部导航栏，卡片式设计
- **电脑端**：多列布局，侧边栏导航，表格+卡片混合设计

### 通用组件
- 顶部导航栏（包含用户信息、消息通知、退出登录）
- 底部导航栏（移动端，主要功能入口）
- 侧边栏导航（电脑端）
- 搜索框
- 筛选器
- 分页器
- 表单输入框
- 按钮（主要、次要、危险）
- 模态框/对话框
- 消息提示
- 加载状态

### 颜色主题
- 主色调：青绿色系（#72C1BB）
- 辅助色：白色、灰色
- 状态色：成功（绿色）、警告（橙色）、错误（红色）

### 交互要求
- 页面间流畅跳转
- 表单验证提示
- 操作反馈（成功/失败提示）
- 数据加载状态
- 空状态展示

## 四、页面优先级

### 高优先级（核心功能）
1. 登录/注册
2. 首页
3. 宠物管理（用户端）
4. 预约管理（用户端+医生端）
5. 预约就诊流程（用户端）
6. 聊天功能（用户端+医生端）

### 中优先级
1. 健康监测/评估
2. 诊断记录
3. 宠物日志
4. 药品管理（医生端）
5. 开具药品（医生端）

### 低优先级
1. 统计页面
2. 健康指南
3. 健康标准
4. 更多功能页面

---

## 五、特殊说明

1. **响应式设计**：所有页面需要同时适配移动端和电脑端
2. **导航方式**：移动端使用底部导航，电脑端使用侧边栏导航
3. **数据展示**：列表页面需要支持搜索、筛选、分页
4. **表单页面**：需要清晰的表单布局和验证提示
5. **聊天功能**：需要支持文字、表情、图片消息
6. **统计页面**：需要图表展示（柱状图、折线图等）

请根据以上功能点，为**用户端**和**医生端**分别设计**移动端**和**电脑端**的原型图。每个页面只需展示主要布局和核心功能元素，不需要过于详细的交互细节。
